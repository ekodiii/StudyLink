# StudyLink

Canvas assignment sync app for study groups.

## Stack
- **Backend**: FastAPI + SQLAlchemy async + PostgreSQL
- **Frontend**: Vanilla JS served as static files from `backend/frontend/`
- **Auth**: Google OAuth
- **Extension**: Chrome extension syncs Canvas data

## Project Structure
- `backend/app/` — FastAPI app (routers, models, schemas, core)
- `backend/frontend/` — HTML/CSS/JS frontend (served by FastAPI)
- `backend/alembic/` — Database migrations (hand-written, not autogenerated)
- `extension/` — Chrome extension

## Database Migrations
- Migrations are in `backend/alembic/versions/`, numbered 001-004
- Migrations are **hand-written** following the existing pattern (not autogenerated)
- **Alembic upgrades run automatically on push to git** — no manual `alembic upgrade head` needed

## Key Conventions
- Frontend is in `backend/frontend/`, NOT a separate `frontend/` directory
- All frontend state is managed with vanilla JS globals (no framework)
- Collapsible sections default to collapsed with inline status summaries
- Client-side overdue detection: `isOverdue()` checks if assignment is past due and unsubmitted/missing
